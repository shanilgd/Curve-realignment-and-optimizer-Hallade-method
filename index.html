User Manual

Hallade Curve Realignment

1. Enter the number of stations before, on, and after the curve.

2. Click Generate Grid to create the input table.

3. Input the Existing Versines and optionally Proposed Versines.

4. Specify any inward/outward slew limits for specific stations.

5. Click Optimize Slew to automatically calculate the best realignment.


Note: You can also import data directly using the Import Excel button using the provided template.

Version History

v1.0.8 - Completely rewrote the optimization engine to use Quadratic Programming (QP) with Linear Constraints. This exponentially speeds up processing for long curves (100+ stations) and eliminates the "sharp spikes" in proposed versines caused by the previous coordinate descent algorithm, resulting in a perfectly smooth and mathematically sound curve.

v1.0.7 - Relocated update notification to bottom-center with high-contrast styling for better visibility. Fixed Excel import to properly display versines with 0 value instead of blank cells.

v1.0.6 - Fixed updater scoping bug causing fatal silent errors. Added global UI toast notification system for instant update feedback.

v1.0.5 - UI Refinements, Earthwork UI Matching, Progress bars, and new 3D Icon.

v1.0.4 - Fixed Hallade mathematical shift bug in Second Summation calculation.

v1.0.3 - UI refinements, auto-updater implementation, and bug fixes.

v1.0.0 - Initial Release. Optimization algorithm using SLSQP.
